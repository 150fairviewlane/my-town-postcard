I want to build a template transformation system for the “Surprise Me” option in my ad generator. Instead of asking Grok to invent a layout from scratch, Surprise Me will pick one of the existing named templates, apply a conservative color and layout transformation using Sharp, and send the modified template image to Grok exactly like a named template.

This session is exploration and testing only. Do not wire anything into the live ad generator yet. Build a standalone test script that applies transformations to each template image and saves the results so I can visually inspect them before deciding which palettes to keep.

# ==============================================================
STEP 4 — Define CONSERVATIVE_PALETTES

Use these conservative transformations only. The goal is
“clearly different color story” not “unrecognizable.”
All operations use Sharp’s built-in methods only.

const CONSERVATIVE_PALETTES = [
{
name: “cool-shift”,
description: “Shifts warm tones toward cool blues/teals. Good for warm templates like parchment and made-fresh.”,
operations: [
// Rotate hue by 150 degrees — warm orange/brown becomes blue/teal
// Conservative: only 150 degrees, not full 180 inversion
{ type: “modulate”, options: { hue: 150, saturation: 0.9, brightness: 1.0 } }
]
},
{
name: “warm-deepen”,
description: “Deepens and warms the palette. Good for cool templates like health-wellness and neighborhood-pro.”,
operations: [
// Shift hue toward warmer tones, slightly increase saturation
{ type: “modulate”, options: { hue: -40, saturation: 1.15, brightness: 0.95 } }
]
},
{
name: “rich-dark”,
description: “Darkens overall palette for a premium moodier look. Works on all templates.”,
operations: [
// Reduce brightness moderately, boost saturation slightly
{ type: “modulate”, options: { hue: 0, saturation: 1.1, brightness: 0.75 } }
]
},
{
name: “fresh-light”,
description: “Lightens and desaturates slightly for a fresh clean look. Works on all templates.”,
operations: [
// Increase brightness slightly, reduce saturation for airy feel
{ type: “modulate”, options: { hue: 0, saturation: 0.75, brightness: 1.2 } }
]
},
{
name: “complementary”,
description: “Shifts to complementary color — most dramatic but still recognizable. Use carefully.”,
operations: [
// 180 degree hue rotation — true complementary colors
// Most dramatic of the conservative options
{ type: “modulate”, options: { hue: 180, saturation: 0.85, brightness: 1.0 } }
]
},
{
name: “muted-mono”,
description: “Desaturates heavily for near-monochromatic look with subtle tint remaining.”,
operations: [
// Heavy desaturation — not full greyscale, just muted
{ type: “modulate”, options: { hue: 0, saturation: 0.25, brightness: 1.05 } }
]
},
];

# ==============================================================
STEP 5 — Implementation notes for the test script

Sharp modulate usage for reference:
sharp(inputBuffer)
.modulate({ hue: 150, saturation: 0.9, brightness: 1.0 })
.flop()  // horizontal flip — add this for flipped variants
.png()
.toBuffer()

For non-flipped variants, omit .flop()

Process each template × each palette × 2 (normal + flipped)
Total output files: 5 templates × 6 palettes × 2 flips = 60 images

This is the inspection set. We will visually review all 60
and select which palette+flip combinations look best per
template before any of this goes into production.
