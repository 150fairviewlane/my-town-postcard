This task replaces the entire variant system with a cleaner approach: show only unused templates in the picker for each postcard side. If a template is already used by a reserved or paid spot on the same side of the same postcard, remove it from the options presented to the next advertiser.

Read all relevant files before making any changes. Confirm the plan before writing code. Build in the order listed below.

# ==============================================================
PART 1 — New API endpoint: GET /api/campaigns/:campaignId/used-templates

Add a new route in the API server that returns which template keys are already in use on each side of a postcard for a given campaign.

Request: GET /api/campaigns/:campaignId/used-templates

Response shape:
{
“front”: [“parchment-classic”, “made-fresh”],
“back”:  [“neighborhood-pro”]
}

Implementation:

- Query the spots table for all spots where:
  - campaignId matches the route param
  - status is “reserved” OR “paid”
  - templateData is not null
- Parse the templateData JSON for each spot and extract the “template” key
- Group by the spot’s “side” column (front/back)
- Return the two arrays of template keys (deduplicated)

This endpoint requires no authentication beyond what other campaign
endpoints use — match the existing auth pattern for campaign queries.

# ==============================================================
PART 2 — Template picker UI: filter out used templates

In the template picker component (wherever the 6 template options
are displayed to the advertiser during ad creation):

1. When the picker opens for a spot, call the new endpoint:
   GET /api/campaigns/:campaignId/used-templates
   Pass the campaignId and the current spot’s side (front/back).
1. Filter the displayed template options to exclude any template
   key already present in the used list for that side.
1. If a template is unavailable, either:
- Hide it entirely from the picker (preferred), OR
- Show it grayed out with a tooltip:
  “Already used on this side of the postcard”
  Use whichever approach fits the existing picker UI better.
1. If ALL templates are already used on that side (only possible
   when there are more spots than templates), show all templates
   with a notice: “All styles are in use on this side. Your ad
   will still look unique based on your business content and photos.”
1. The current spot’s own template (if already generated) should
   NOT count as taken — only OTHER spots’ templates are excluded.
   Filter using ne(spotsTable.id, currentSpotId) in the query.
