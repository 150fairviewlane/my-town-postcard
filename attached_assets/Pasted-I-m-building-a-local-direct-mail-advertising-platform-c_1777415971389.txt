I’m building a local direct-mail advertising platform called LocalSpot Mailer. It’s a 9x12 co-op postcard business where local businesses buy ad spots on a shared postcard mailed to 5,000 homes via USPS EDDM. I already have a working PostcardSpotPicker.jsx component. Now I need you to build out the full application around it.
Tech stack: React (Vite), Node.js + Express backend, PostgreSQL database, Stripe for payments, Cloudinary for ad image uploads, Resend for transactional email.
Please build the following in order:
SECTION 1 - DATABASE SCHEMA: Create a schema.sql file with these tables. Table “campaigns” with fields: id, name, territory, zip_code, mail_date, homes_count, status (active/closed/mailed), created_at. Table “spots” with fields: id, campaign_id, size (large/medium/small), grid_area, price, category_lock, status (available/reserved/paid), business_name, business_category, contact_email, contact_phone, ad_file_url, created_at. Table “orders” with fields: id, spot_id, stripe_payment_intent_id, business_name, contact_email, contact_phone, amount_paid, paid_at, ad_status (pending/uploaded/approved).
SECTION 2 - EXPRESS BACKEND: Build a REST API in server/index.js with these endpoints. GET /api/campaigns/active returns the active campaign with all its spots. POST /api/spots/:id/reserve reserves a spot with name, category, email, phone, checks category is not already taken, and returns spot data. POST /api/checkout/create-payment-intent creates a Stripe PaymentIntent for the spot price. POST /api/checkout/confirm marks spot as paid after Stripe confirms, creates order record, and sends confirmation email via Resend. POST /api/spots/:id/upload-ad accepts a file upload of PNG or PDF max 10MB, uploads to Cloudinary, and saves URL to spot record. GET /api/admin/orders returns all orders with spot and campaign info protected with a simple admin token in the header. POST /api/admin/spots/:id/approve marks ad as approved for admin only. Use dotenv for environment variables. Required env vars are DATABASE_URL, STRIPE_SECRET_KEY, CLOUDINARY_URL, RESEND_API_KEY, and ADMIN_TOKEN.
SECTION 3 - STRIPE CHECKOUT FLOW: Create src/CheckoutPage.jsx that shows a summary of the reserved spot including size, price, and business name, embeds Stripe Elements for card payment, redirects to a confirmation page on success, and shows an error message with retry option on failure.
SECTION 4 - AD UPLOAD PAGE: Create src/UploadAdPage.jsx shown after payment succeeds. It lets the customer upload their ad file as PNG or PDF with a size guide for their spot dimensions, or lets them check a box saying “I need you to design my ad for me” which sets ad_status to design_requested, and shows a success message when done.
SECTION 5 - ADMIN DASHBOARD: Create src/AdminDashboard.jsx as a password-protected page at /admin. It shows all spots for the active campaign in a table with columns for business name, category, size, price, paid status, and ad status. It shows total revenue collected vs total spots. It has an Approve Ad button per row that calls the approve endpoint. It has a Download Print-Ready PDF button as a placeholder. Login uses a hardcoded admin password stored in env.
SECTION 6 - REACT ROUTER SETUP: Wire up these routes in src/App.jsx. Route / loads PostcardSpotPicker as the home page. Route /checkout/:spotId loads CheckoutPage. Route /upload/:spotId loads UploadAdPage. Route /confirmation loads a simple thank-you page. Route /admin loads AdminDashboard.
SECTION 7 - EMAIL TEMPLATES: Create server/emails.js with two functions. Function sendReservationConfirmation(order) sends to the customer confirming their spot reservation with their business name, spot size, price, and a link to upload their ad. Function sendAdminNewOrder(order) sends to the admin email notifying of a new paid order.
SECTION 8 - ENVIRONMENT VARIABLES: Create a .env.example file listing all required variables with placeholder values and comments explaining where to get each one such as Stripe dashboard and Cloudinary.
Important notes: Keep the existing PostcardSpotPicker.jsx exactly as-is and just connect it to the real backend instead of local state. All prices are in cents for Stripe so Large is 39900, Medium is 29900, and Small is 19900. Enforce the one business per category per campaign rule on the backend not just the frontend. Use mobile-responsive design throughout. Use the brand color #991b1b for buttons and accents throughout.
