STEP 3 — SIMPLIFIED TERRITORY BUILDER
════════════════════════════════════════════════

Replace getTerritoryForZip() with:

  async function getTerritoryForLocation(
    lat: number,
    lng: number,
    stateAbbr: string,
    zipCode?: string
  ): Promise<{
    type: 'existing' | 'proposed' | 'unavailable'
    territory?: any
    proposal?: TerritoryProposal
    message?: string
  }>

  STEP A — Check for existing territory near this location
    Query territories table for any territory whose centroid
    is within 25 miles of (lat, lng) AND state = stateAbbr
    If found and status = 'available' or 'taken':
      return { type: 'existing', territory }

  STEP B — Find 4 hub cities
    Call findCandidateHubs(lat, lng, stateAbbr)
    Call selectBestHubs(candidates, lat, lng)

  STEP C — If fewer than 4 hubs qualify, expand radius
    Increase search radius by 15 miles and try again
    Keep expanding up to 3 times (max 85 mile radius)
    Accept territory with minimum 3 qualifying hubs
    If still fewer than 3: return { type: 'unavailable',
      message: 'No viable territory found in this area.' }

  STEP D — Build and return proposal
    Generate territory name from hub city names
    Generate slug
    Calculate centroid as average lat/lng of all hubs
    return { type: 'proposed', proposal }

    Do NOT save to territories table yet.
    Save only when dealer completes Stripe payment.

STEP 4 — INSTANT ACTIVATION ON PAYMENT
════════════════════════════════════════════════

When dealer completes Stripe payment (webhook
checkout.session.completed):

  1. Save territory to territories table:
       id = generated (state abbreviation + sequential number
            e.g. TN-001, OH-001)
       name = proposal.proposedName
       state = stateAbbr
       counties = hub city county names array
       zone_note = hub city names joined by ', '
       households = sum of hub catchment households
       centroid_lat, centroid_lng from proposal
       status = 'taken'
       dealer_id = paying dealer's id

  2. Create landing_pages record, is_published = true
  3. Create 15 ad_slots with status = 'available'
  4. Send dealer confirmation email
  5. Done — no approval needed, no waiting

Remove all pending_review logic.
Remove the admin approval queue routes and page.
Remove the admin notification email for new proposals.

STEP 6 — MAP DISPLAY — SAME FOR ALL 50 STATES
════════════════════════════════════════════════

The map must work identically for every state.
No Georgia-specific code. No special cases.

ON PAGE LOAD:
  Load all territories from the territories table
  (initially empty — map starts with no colored polygons)

  For each territory in the table:
    Load county GeoJSON for that territory's state
    using the existing /api/counties-geojson?state=XX endpoint
    Find the county features matching the territory's counties
    Color them: green (available) or red (taken)
    Make them clickable

  Cache GeoJSON per state in memory — only fetch each
  state's GeoJSON once per session

WHEN A PROPOSAL IS RETURNED (type='proposed'):
  Load county GeoJSON for the proposal state if not cached
  Find the 4 hub county features by county FIPS
  Color them amber (#F59E0B)
  Make each polygon clickable
  Clicking a polygon selects that hub in the left panel

CLICKING ANY TERRITORY POLYGON (existing or proposed):
  Updates the left panel with that territory's details
  Shows name, hub cities, market tier, claim button
  Same behavior for every state — no special cases

POLYGON COLORS:
  Green (#86efac border #16a34a) — available territory
  Red (#fca5a5 border #dc2626) — taken territory
  Amber (#fde68a border #d97706) — current proposal
  Grey (default) — no territory assigned

REMOVE:
  All Georgia-specific GeoJSON loading code
  Any code that treats Georgia differently from other states
  The radius circle fallback — never show circles, always
  show county polygons
