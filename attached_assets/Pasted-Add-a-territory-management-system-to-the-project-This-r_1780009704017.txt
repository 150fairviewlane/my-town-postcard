Add a territory management system to the project. This replaces the existing zip-code-based territory system. Follow these instructions carefully.


════════════════════════════════════════════════
OVERVIEW
════════════════════════════════════════════════

The new system assigns dealer territories based on counties rather than
zip codes. Each territory is a named group of one or more contiguous
counties. Territories are non-overlapping by definition since every
county belongs to exactly one territory.

The system has two parts:
  1. Admin Territory Manager — Dave defines and manages territories
  2. Dealer Signup — dealers browse available territories and claim one

The system is designed to support multiple states. Georgia is the only
active state for now. Adding new states requires only adding county data
and territory definitions for that state — no structural changes needed.


════════════════════════════════════════════════
CHANGE 1 — DATABASE TABLES
════════════════════════════════════════════════

Create two new database tables:

TABLE: territories
  id              — string, primary key (e.g. "GA-001")
  name            — string (e.g. "White / Habersham Counties")
  state           — string, 2-letter code (e.g. "GA")
  counties        — JSON array of county name strings
  households      — integer, estimated total households
  zones           — integer, always 4
  status          — string: "available", "pending", or "taken"
  dealer_id       — integer or null, foreign key to dealers table
  created_at      — timestamp

TABLE: dealer_territory_claims
  id              — integer, primary key
  dealer_id       — integer, foreign key to dealers table
  territory_id    — string, foreign key to territories table
  claimed_at      — timestamp
  status          — string: "pending", "active", "cancelled"

Seed the territories table with the Georgia territories defined in the
TERRITORY_DB object inside territory-manager.html. There are 12 starter
territories defined there — insert all of them on first run.


════════════════════════════════════════════════
CHANGE 2 — API ROUTES
════════════════════════════════════════════════

Create these new API routes:

GET /api/territories?state=GA
  Returns all territories for a given state as JSON array
  Each territory object includes: id, name, state, counties,
  households, zones, status, dealer_id

POST /api/territories
  Creates a new territory (admin only, requires auth)
  Body: { name, state, counties, households }
  Returns the created territory object

PUT /api/territories/:id
  Updates a territory (admin only, requires auth)
  Body: any subset of { name, counties, households, status, dealer_id }
  Returns the updated territory object

DELETE /api/territories/:id
  Deletes a territory (admin only, requires auth)
  Only allowed if status is "available"

POST /api/territory-claims
  Creates a dealer territory claim
  Body: { dealer_id, territory_id, dealer_name, dealer_email, dealer_phone }
  Sets territory status to "pending"
  Returns the claim object
  In production: trigger Stripe checkout for $99/month subscription

GET /api/territories/:id/counties
  Returns the list of counties in a territory
  Used for EDDM zone planning by admin

CHANGE 5 — MAP UPGRADE NOTE
════════════════════════════════════════════════

The current map uses simplified rectangle shapes for counties as
placeholders. For a production-quality map, replace these with real
Georgia county GeoJSON from the US Census Bureau TIGER/Line files:
  https://www.census.gov/geographies/mapping-files/time-series/geo/tiger-line-file.html

This is optional for the initial launch but strongly recommended before
showing the map to dealers. The rectangles are functional for admin use.

To upgrade the map:
  1. Download Georgia county GeoJSON
  2. Add a script tag to load D3.js from cdnjs.cloudflare.com
  3. Replace the renderMap() function to use D3 geo projection
  4. The rest of the code (tooltips, selection, coloring) stays the same

This upgrade does not require any database or API changes.
