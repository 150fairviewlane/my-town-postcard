WHAT CHANGED
════════════════════════════════════════════════

This update fixes font sizing and element positioning throughout the ad canvas.

ROOT CAUSE OF ALL PREVIOUS ISSUES:
Every font was using vw units (viewport width), which reference the full
browser width of ~1300px on desktop. The ad canvas is only ~480px wide,
so all fonts were rendering at roughly 2.7x the intended size. This caused
the company name to overflow, the tagline to be enormous, the menu items
to collide with the coupon, and the coupon to render too large.

FIXES APPLIED:
All font sizes now use cqw (container query width) or min() with a px cap,
so they scale relative to the ad canvas width, not the browser viewport.

Specific changes:
Business name Line 1: was 7.8vw, now min(9.5cqw, 46px)
Business name Line 2: was 9.5vw, now min(11cqw, 54px)
Tagline: was 3.9vw, now min(4.8cqw, 23px)
Menu item names: was 2.1vw, now min(2.5cqw, 12px)
Menu item prices: was 2.1vw, now min(2.5cqw, 12px)
Coupon amount: was 5.8vw, now min(6.5cqw, 32px)
Coupon item: was 4vw, now min(4.8cqw, 23px)
Coupon fine print: was 1.2vw, now min(1.8cqw, 9px)
Phone number: was 5vw, now min(5.5cqw, 27px)
Address: was 1.3vw, now min(1.8cqw, 9px)

Menu block vertical position adjusted:
top changed from 50% to 49% so row centers align with checkmark circles
at 53.03%, 60.69%, 67.98%, 75.64% of card height
