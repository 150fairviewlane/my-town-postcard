CHANGE 1 — FIX THE TERRITORY DETAIL PANEL
════════════════════════════════════════════════

On the /find-territory page, the selected territory detail panel
currently shows the counties list twice and no city information.

Find where the detail panel HTML is rendered (in territory-finder.html
or the frontend component that renders the selected territory detail).

Make these changes:

1. Remove the duplicate counties line — show counties only once,
   as a small subdued label under the territory name.

2. Add a "Key Cities" row that displays the zone_note field from
   the territory record. Label it "Key Cities" and show the value
   as a comma-separated list.

3. The final detail panel layout should be:

   TERRITORY NAME (large, bold)
   Counties: Lumpkin · White  (small, subdued)

   Key Cities         Dahlonega, Cleveland, Helen, Suches
   Households         ~22,000
   Status             Available
   Monthly Fee        $99 / month

   [CLAIM THIS TERRITORY button]

4. In the nearby territory list cards on the left sidebar, also add
   the zone_note cities as a third line under the county names,
   in a smaller lighter font. Keep it to one line — truncate with
   ellipsis if too long.

The zone_note field is already returned by GET /api/territories?state=GA
so no API changes are needed — just display it in the frontend.
