STEP 2 — FIX COMMON PDF RESPONSE ISSUES:
The blank tab is usually caused by one of these. Fix all of them:

A) If using pdfkit's pipe/stream API, the PDF may not be fully generated before the response is sent. Switch to collecting the PDF into a Buffer first, then send it all at once:

  const pdfBuffer = await new Promise<Buffer>((resolve, reject) => {
    const doc = new PDFDocument({ size: [842, 595], margin: 0 }); // 12x9 at 72dpi = 864x648, use A4 landscape as close match or use custom
    const chunks: Buffer[] = [];
    doc.on('data', chunk => chunks.push(chunk));
    doc.on('end', () => resolve(Buffer.concat(chunks)));
    doc.on('error', reject);
    // ... add image to doc ...
    doc.end();
  });
  
  res.setHeader('Content-Type', 'application/pdf');
  res.setHeader('Content-Disposition', 'inline; filename="postcard.pdf"');
  res.setHeader('Content-Length', pdfBuffer.length);
  res.setHeader('Cache-Control', 'no-store');
  res.end(pdfBuffer);

Note: use 'inline' not 'attachment' for Content-Disposition — inline tells Safari to display it in the PDF viewer rather than download it, which is what works on iOS.

B) The PDF page size must match the image dimensions. For a 12x9 inch postcard at 72 PDF points per inch:
  width = 12 * 72 = 864 points
  height = 9 * 72 = 648 points
  const doc = new PDFDocument({ size: [864, 648], margin: 0, autoFirstPage: true });

C) When embedding the composited image in pdfkit, the image must fill the entire page:
  doc.image(compositedBuffer, 0, 0, { width: 864, height: 648 });

D) The composited sharp image must be output as JPEG or PNG buffer before passing to pdfkit:
  const compositedBuffer = await canvas.jpeg({ quality: 92 }).toBuffer();
  // NOT .toFile() — must be a Buffer
