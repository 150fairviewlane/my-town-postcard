Build a dynamic multi-territory landing page system. When a dealer's subscription activates, the system automatically creates a landing page for their territory at a unique URL. The page is identical to the existing Habersham page in design and content, with only territory-specific fields replaced. Businesses can purchase ad slots directly via Stripe.


════════════════════════════════════════════════
OVERVIEW
════════════════════════════════════════════════

The existing Habersham page at / (or /habersham) is the reference design.
Do not modify it. Instead:

1. Convert it into a dynamic template that renders any territory
2. Auto-create a landing page record when a dealer's Stripe
   subscription activates
3. Serve each territory at /[slug] e.g. /white-county, /hall-county
4. Show 15 ad slots per page (split Front Side / Back Side)
5. Let businesses purchase slots via Stripe Checkout
6. Give dealers a simple portal to share their page and track sales


════════════════════════════════════════════════
CHANGE 1 — DATABASE TABLES
════════════════════════════════════════════════

Create these tables. Use the same database/ORM already in the project.

TABLE: landing_pages
  id                  serial primary key
  territory_id        text references territories(id)
  slug                text unique not null
  dealer_id           integer references dealers(id)
  is_published        boolean default false
  mailing_season      text default 'Summer 2026'
  mailing_month       text default 'third week of June 2026'
  target_homes        integer default 5000
  created_at          timestamp default now()

TABLE: ad_slots
  id                  serial primary key
  landing_page_id     integer references landing_pages(id)
  slot_number         integer not null (1–15)
  side                text not null ('front' or 'back')
  size                text not null ('small','medium','large','extra-large')
  price               integer not null (in cents: 19900/29900/39900/49900)
  position            text (e.g. 'top-left', 'center', 'bottom-right')
  status              text default 'available' ('available','reserved','sold')
  advertiser_name     text
  advertiser_email    text
  advertiser_phone    text
  business_category   text
  ad_image_url        text
  stripe_session_id   text
  stripe_payment_id   text
  purchased_at        timestamp
  created_at          timestamp default now()
  UNIQUE (landing_page_id, slot_number)

TABLE: advertisers
  id                  serial primary key
  landing_page_id     integer references landing_pages(id)
  business_name       text not null
  contact_name        text
  email               text not null
  phone               text
  business_category   text
  stripe_customer_id  text
  created_at          timestamp default now()

The dealers table should already exist from the territory claim system.
Add these columns if not already present:
  stripe_customer_id  text
  stripe_subscription_id text
  subscription_status text default 'inactive'
  landing_page_id     integer references landing_pages(id)

CHANGE 3 — DYNAMIC LANDING PAGE TEMPLATE
════════════════════════════════════════════════

Create a new route: GET /:slug

This route:
1. Looks up the landing_page record by slug
2. If not found or is_published = false: return 404
3. Joins to territories table to get: name, zone_note, households
4. Joins to dealers table to get dealer contact info
5. Fetches all 15 ad_slots for this landing_page_id
6. Renders the template with territory-specific data injected

The template is identical to the existing Habersham page with these
fields replaced dynamically:

FIELD REPLACEMENTS (all instances throughout the page):

  "Habersham County" → territory.name
  "Habersham" (standalone) → territory.name (shortened if needed)

  Hero subheadline:
    "5,000 Clarkesville, Demorest, Cornelia and Alto homes"
    → "{{target_homes | number}}, {{city_list}} homes"
    where city_list = territory.zone_note split by comma,
    first 4 cities joined with commas and "and" before last

  How It Works Step 3:
    "5,000 postcards printed and delivered to Habersham homes via USPS"
    → "{{target_homes}} postcards printed and delivered to
       {{territory.name}} homes via USPS"

  Pricing section mailing card:
    "Summer 2026 Mailing"
    → "{{landing_page.mailing_season}} Mailing"

    "Timed to reach 5,000 Habersham County homes during peak local
     shopping season — targeted for the third week of June 2026"
    → "Timed to reach {{target_homes}} {{territory.name}} homes
       during peak local shopping season — targeted for
       {{landing_page.mailing_month}}"

  Targeted Areas card:
    "We focus on specific Habersham County neighborhoods where
     your customers already live"
    → "We focus on specific {{territory.name}} neighborhoods where
       your customers already live"

  Page title tag:
    "My Town Postcard"
    → "{{territory.name}} Postcard | My Town Postcard"

AD SLOT GRID:
  Replace the static Habersham ad grid with a dynamic grid built
  from the 15 ad_slot records. Each slot renders as either:

  A) SOLD/RESERVED: shows the advertiser's ad image if uploaded,
     or a placeholder with business name and category

  B) AVAILABLE: shows the green "+" button with:
     - Ad size label (e.g. "Extra Large Ad")
     - Price (e.g. "$499")
     - Ad dimensions (e.g. "4" × 5"")
     - "Reserve This Spot" button that triggers Stripe Checkout
     - "Reaches {{target_homes}} local homes" subtext

  Keep the Front Side / Back Side tab toggle exactly as it is now,
  with live counts of sold slots per side.

CHANGE 4 — AD SLOT PURCHASE VIA STRIPE
════════════════════════════════════════════════

When a business clicks "Reserve This Spot":

Step 1 — Show a modal or inline form collecting:
  Business Name *
  Contact Name *
  Email *
  Phone *
  Business Category * (dropdown — same 32 categories as the ad studio)

Step 2 — On form submit, POST to /api/slots/:slotId/checkout
  This endpoint:
  a. Validates the slot is still available
  b. Creates or finds a Stripe Customer with the email
  c. Creates a Stripe Checkout Session:
       mode: 'payment'
       line_items: [{ price_data: { currency: 'usd',
         unit_amount: slot.price,
         product_data: { name: slot.size + ' Ad — ' + territory.name,
           description: slot.size + ' ad space on the ' +
             territory.name + ' postcard, reaching ' +
             target_homes + ' local homes' }},
         quantity: 1 }]
       success_url: '/[slug]/success?session_id={CHECKOUT_SESSION_ID}'
       cancel_url: '/[slug]'
       metadata: { slotId: slot.id, landingPageId: slot.landing_page_id }
  d. Marks slot as 'reserved' in the database
  e. Returns { url: checkoutUrl } to redirect the browser

Step 3 — Stripe webhook at /api/webhooks/stripe handles
  checkout.session.completed:
  a. Reads metadata.slotId from the session
  b. Marks slot status = 'sold'
  c. Stores stripe_payment_id and purchased_at on the slot
  d. Sends confirmation email to advertiser (if email sending exists)
  e. Notifies dealer (if email sending exists)

Step 4 — Success page at /[slug]/success:
  Shows "Your spot is reserved!" confirmation with:
  - Territory name
  - Slot size and price paid
  - What happens next (dealer will contact to create your ad)
  - Dealer contact info

CHANGE 5 — AUTO-CREATE LANDING PAGE ON DEALER ACTIVATION
════════════════════════════════════════════════

When a dealer's Stripe subscription becomes active (webhook event
customer.subscription.updated with status = 'active', or
checkout.session.completed for a dealer subscription purchase):

Automatically run this sequence:

1. Find the dealer's territory_id from their record
2. Look up the territory to get name and zone_note
3. Generate a slug from the territory name:
   - Lowercase
   - Replace spaces with hyphens
   - Remove special characters
   - Examples: "White / Habersham Counties" → "white-habersham"
     "Hall County" → "hall-county"
     "Alpharetta / Milton / Roswell" → "alpharetta-milton-roswell"
4. Check slug is unique — append -2, -3 etc if collision
5. INSERT into landing_pages:
   territory_id, slug, dealer_id, is_published = true,
   mailing_season = 'Summer 2026',
   mailing_month = 'third week of June 2026',
   target_homes = 5000
6. INSERT all 15 ad_slot records with status = 'available'
7. UPDATE dealers SET landing_page_id = new_id
8. Log "Landing page created: /[slug]" to console

The page is immediately live and accessible at /[slug].
