Change this from a blocking requirement to a dismissible, non-blocking hint.

**Find the rotate/orientation overlay** — likely a fixed-position full-screen div triggered by a `@media (orientation: portrait)` CSS rule or a JS `orientationchange`/`matchMedia` listener, probably styled with something like `position:fixed;inset:0;z-index:9999` to cover the whole page.

Users in portrait mode should be able to use the ad picker, scroll through templates, and proceed through the rest of the flow exactly as if in landscape — just on a narrower layout.

Use a `@media (max-width: ...)` rule scoped to `.tmpl-grid` rather than touching unrelated layout.
